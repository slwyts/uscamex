# USCAMEX 开发文档

> **总原则：所有规则与参数均不写死，全部通过后台（Admin 面板 与 合约的管理员函数）可调整。部署时只需配置一个项目方 owner 地址，其余资金池由系统自动部署并统一由 owner 控制。**

---

## 零、架构决策与技术澄清

### 0.1 AMM 选型：PancakeSwap

- 基于 **PancakeSwap V2** 构建，不自建 AMM。
- LP Token 是真正的 PancakeSwap LP Token。

### 0.2 代币拦截机制（核心技术）

由于 LP 是 PancakeSwap 原生的，用户可以直接去 PancakeSwap 操作，因此需要在代币合约的 `_transfer` 中实现拦截逻辑：

- **检测转账方向**：识别 `pair → router → user` 的调用链。
- **K 值变化检测**：判断是买入还是卖出。
- **负债表机制**：在交易调用链的下次转账时（`pair → router → user` 过程中），通过 transfer 钩子扣除应扣代币，实现税收和销毁逻辑。
- 撤 LP 时同理：检测到从 pair 流出的代币转账，将代币部分拦截销毁，仅放行 BNB。

### 0.3 奖励发放机制：懒加载（Lazy Evaluation）

- 静态和动态奖励**不主动推送**，采用懒加载方式。
- 用户进行任何交互时（claim、入金、撤出等），合约自动结算其应得奖励。
- 如果有合适的预言机 / Keeper，也可考虑作为辅助触发。
- **实现时需特别注重代码逻辑正确性。**

### 0.4 节点定义

- 节点 = 入金用户，入金金额即为节点权重（0.1~5 BNB，管理员可调范围）。
- 不同入金金额的节点拥有不同权重，按权重加权平分节点分红。
- 后台可新增 / 移除节点地址。

### 0.5 推荐关系绑定方式

- 给目标地址转账 **正好 10 枚整数代币**（排除合约地址）即绑定该地址为自己的上级。
- 绑定时检查：被绑定的上级必须自身也已绑定了上级（防止绑定到不存在的推荐链）。
- **合约 owner 为初始根上级**（推荐链的顶端）。
- 总量 1%（1000 万枚）用于绑定操作。

## 二、交易税机制

### 2.1 买入税 — 3%（可后台调整）

| 比例 | 去向 | 币种 |
|------|------|------|
| 1% | LP 建设者分红池 | 项目代币 |
| 2% | 回购销毁钱包 | BNB |

- **前期不开放买入单**，买入开关由后台控制。

### 2.2 卖出税 — 10%（可后台调整）

| 比例 | 去向 | 币种 |
|------|------|------|
| 3% | LP 建设者分红池 | 项目代币 |
| 3% | 生态建设基金 | BNB |
| 4% | 回购销毁钱包 | BNB |

- **卖出时，扣完税后剩余的项目代币全部进入黑洞地址销毁。**
- **卖出税超出 10% 的部分**：若后台将卖出税设为 > 10%（如 12%），多出的 2% 代币直接进入 dead 黑洞地址销毁。
- 举例：用户卖出 100 枚代币 → 3% 代币进入分红池 + 3% BNB 进生态基金 + 4% BNB 进回购销毁钱包 + 剩余代币全部销毁。

## 三、核心资金池说明

| 资金池 | 用途 | 管理方式 |
|------|------|----------|
| **LP 建设者分红池** | 存放项目代币，用于发放静态 / 动态奖励 | 系统自动部署，owner 可提取 |
| **生态建设基金池** | 存放生态税产生的 BNB | 系统自动部署，owner 可提取 |
| **回购资金池** | 作为回购资金落地池 | 系统自动部署，owner 可提取 |

- 三个资金池默认都由系统自动部署为 owner 统一控制的金库合约。
- 项目方部署时只需要提供一个 owner 地址，不需要额外准备三个外部钱包。
- 分红池主要持有项目代币；生态基金池和回购资金池主要承接 BNB。
- 自动回购执行时，系统优先消耗合约内部记录的 `buybackReserve`，需要人工转出时再提到回购资金池。

## 五、回购销毁机制

- **触发方式**：项目方从后台手动启动 / 关闭。
- **执行逻辑**：启动后，每分钟使用系统内部记录的回购储备 BNB 从 LP 底池买入项目代币并销毁。
- **停止条件**：回购储备耗尽 **或** 项目方手动关闭。

| 参数 | 默认值 | 可调 |
|------|--------|------|
| 每分钟回购额度 | 0.1 BNB | 是（如可调为 0.01 BNB） |

## 六、用户入金（添加 LP）

### 6.1 基本规则

- 入金币种：**BNB**
- 入金方式：直接往代币合约打入 BNB，合约自动组建 LP
- 入金范围：0.1 ~ 5 BNB（**后台可调区间，如设为 0.1~1 则用户只能在此范围内入金**）

### 6.2 入金分配（以用户入金 1 BNB 为例）

| 比例 | 去向 | 说明 |
|------|------|------|
| 60% (0.6 BNB) | 组建 LP | 50% BNB + 50% 项目代币（用 BNB 从 LP 底池买入代币凑对） |
| 10% (0.1 BNB) | 节点加权平分 | 分给已购节点地址（可后台调比例、增减节点） |
| 10% (0.1 BNB) | 买入代币 → LP 建设者分红池 | 从 LP 底池买入项目代币 |
| 10% (0.1 BNB) | 回购资金池 | 记入系统回购储备 |
| 10% (0.1 BNB) | 直推奖励 | 发给推荐人（可后台调比例，差额归系统回购储备） |

### 6.3 节点销售

- 前期需要先卖节点，需创建一个节点销售入口 / 通道。
- 后台可配置：节点分红比例、新增节点地址、取消节点地址。

## 七、撤出 LP 规则

- 撤出 LP 时，**项目代币全部销毁**（进入黑洞地址），仅返还 BNB 部分。
- 举例：入金 1 BNB → 组建 0.3 BNB + 等值项目代币的 LP → 撤出时仅取回 **0.3 BNB**（即本金的 30%），代币全部销毁。
- 撤出后该 LP 订单 **失效**，静态与动态奖励停止。
- 若想继续领取奖励，需用**同一地址重新添加 LP**。

## 八、出局机制（BNB 金本位）

| 参数 | 默认值 | 可调 |
|------|--------|------|
| 出局倍数 | 3 倍 | 是（后台可调） |
| 出局计算基准 | 入金 BNB 数量 | - |
| 累计范围 | **静态 + 动态** 合计 | - |

- 出局 = 用户累计获得的静态奖励 + 动态奖励的 BNB 等值总额 >= 入金额 x 出局倍数。
- 达到出局条件时：**项目代币全部销毁**，与撤出 LP 规则一致，仅返还 BNB。
- 出局后需 **重新添加 LP** 才能继续拿团队奖励。

---

## 九、静态收益

| 参数 | 默认值 | 可调 |
|------|--------|------|
| 日收益率 | 入金额的 0.8% | 是（后台改后全局生效） |
| 发放频率 | 每 6 小时一次，每日 4 次 | - |
| 结算币种 | 项目代币（按 BNB 等值计算） | - |

- **BNB 金本位**：收益按 BNB 价值计算，以项目代币形式发放。
- 举例：A 入金 1 BNB，日静态 0.8% → 每日获得价值 0.008 BNB 的项目代币。
- 每次发放 = 日收益率 / 4。
- **后台修改静态利率后，新利率立即全局生效，且以新利率继续累计至出局额度。**

## 十一、团队动态奖励（代数奖）

领取条件：根据用户**直推人数**解锁对应代数的奖励。

| 直推人数 | 解锁代数 | 该代奖励比例 |
|----------|----------|--------------|
| 1 人 | 第 1 代 | 10% |
| 2 人 | 第 2 代 | 9% |
| 3 人 | 第 3 代 | 8% |
| 4 人 | 第 4 代 | 7% |
| 5 人 | 第 5 代 | 6% |
| 6 人 | 第 6 代 | 5% |
| 7 人 | 第 7 代 | 5% |
| 8 人 | 第 8 代 | 5% |
| 9 人 | 第 9 代 | 5% |
| 10 人 | 第 10 代 | 5% |

- **奖励基数**：下级对应代数内用户的**静态收益**的百分比。
- 举例：A 直推了 3 人 → A 可拿第 1 代下线静态收益的 10% + 第 2 代的 9% + 第 3 代的 8%。
- 最深 10 代。

## 十二、推荐关系绑定

- 使用总量 1%（1000 万枚代币）用于绑定推荐关系。
- **绑定方式**：用户向目标上级地址转账 **正好 10 枚整数代币**，即自动绑定该地址为自己的上级。
- **排除合约地址**：转账目标为合约地址时不触发绑定逻辑。
- **防孤立检查**：被绑定的上级必须自身已绑定了上级（即推荐链不能断裂）。
- **根节点**：合约 owner 为初始根上级，是推荐链的顶端，无需绑定。

## 十三、前期需求：节点销售官网入金通道

- 在合约开发前，先需要一个简单的官网入金通道用于销售节点。
- 功能：用户通过网页连接钱包，支付 BNB 购买节点。
- 后台管理：查看 / 新增 / 移除节点地址。
